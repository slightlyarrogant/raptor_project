Ogólne informacje
W wersji .NET 5.0 Microsoft zrywa kompatybilność wsteczną z frameworkiem Windows Communication Foundation (WCF). W konsekwencji tego została wycięta cała gałąź System.Web.Services co skutkuje brakiem wsparcia dla rozwiązań opartych o schematy WSDL i protokół SOAP.
Microsoft Windows Compatibility
W celu zachowania komunikacji z zewnętrznymi usługami opartymi na protokole SOAP należy zainstalować zestaw bibliotek Microsoft Windows Compatibility np: poprzez instalacjie NuGet-u.
Visual Studio -> Tools -> Package Manager Console:
Install-Package Microsoft.Windows.Compatibility
Kolejnym krokiem jest ponowne wygenerowanie modeli C# na podstawie plików WSDL. W tym celu wykorzystuje my narzedzie dostarczane przez Microsoft Svcutil.exe w najnowszej wersji (2.0.2 stan na 06.10.2021). Uruchamiamy konsolę Visual Studio Developer Command Prompt i uruchamimy polecenie:
dotnet-svcutil [ścieżka po pliku WSDL] --outputDir [ścieżka do folderu wynikowego]
Jako wynik otrzymujemy gotowy model w postaci pliku .cs. Pełna lista parametrów jest dostepna po wywołaniu
dotnet-svcutil -h
